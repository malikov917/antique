## Findings (Prioritized)
1. **P1: Feed "Inbox" CTA collides with Inbox tab intent**
- Location: Feed + tab bar navigation
- Impact: users and automation can open overlay when intent was tab switch; lowers navigation reliability.
- Evidence: `ios-inbox-ambiguity-failure.png`, `ant57-ios-inbox-overlay.png`
- Follow-up issue: [ANT-58](https://linear.app/antique/issue/ANT-58/mobile-disambiguate-feed-inbox-cta-vs-inbox-tab-navigation)

2. **P1: Profile onboarding and auth-gated controls are unclear pre-auth**
- Location: Profile (pre-auth)
- Impact: first-time users do not get explicit staged path; disabled controls look broken without local explanation.
- Evidence: `ios-profile-preauth.png`, `ant57-ios-profile-tab.png`
- Follow-up issue: [ANT-59](https://linear.app/antique/issue/ANT-59/mobile-improve-profile-onboarding-clarity-and-auth-gated-section-ux)

3. **P2: Feed title selector mismatch increases test fragility (`Live feed` vs `Feed`)**
- Location: Feed assertions in Maestro
- Impact: brittle tests and false-negative automation failures.
- Evidence: `ios-feed-assertion-failure.png`
- Suggested direction: standardize a stable testID for feed root title and use selector-based assertions.

## Top 5 UX Blockers for Conversion/Retention
1. Inbox CTA vs tab ambiguity (P1)
2. Profile onboarding intent clarity gap (P1)
3. Auth-gated actions displayed without inline rationale (P1)
4. OTP terminology too technical for first-run users (P2)
5. Text-selector instability in core tab walkthrough automation (P2)

## Quick Wins (<1 day)
- Assign unique labels and accessibility IDs for Inbox tab and overlay CTA.
- Add pre-auth explanatory copy near each disabled Profile section.
- Replace technical OTP CTA text with user-oriented language (`Send code`, `Continue`).
- Add stable testIDs for tab-bar roots and feed header.

## Deeper Improvements
- Introduce explicit staged Profile onboarding layout (`Sign in -> Complete profile -> Seller application`).
- Add a platform parity checklist to mobile release gating (iOS + Android evidence set required).

## Maestro Regression Checks to Add
- Tab intent check: text/ID selector for Inbox tab must not open feed overlay.
- Profile pre-auth check: explanatory gating copy visible next to locked sections.
- Feed root check: assert by testID, not display text variant.
